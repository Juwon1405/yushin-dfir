# Voiceover assets for the Agentic-DART demo screencast

British-English narration for the SANS FIND EVIL! 2026 demo video.

## What's here

```
docs/voiceover/
├── README.md               (this file)
├── say_voiceover.sh        one-shot synthesis script (macOS)
├── scenes/
│   ├── scene1.txt          cold open
│   ├── scene2.txt          one-line install
│   ├── scene3.txt          architectural boundary / bypass tests
│   ├── scene4.txt          end-to-end case study run
│   ├── scene5.txt          v0.6.1 new capabilities (DNS tunneling)
│   └── scene6.txt          closing slate
└── out/                    .aiff and .mp3 outputs (generated by the script)
```

## Quick start

On your **macOS host** (not inside the SIFT VM):

```bash
cd ~/agentic-dart
bash docs/voiceover/say_voiceover.sh
```

That produces six `.aiff` files and six `.mp3` files in `docs/voiceover/out/`,
one per scene, narrated by `Daniel` (formal British male, built into macOS).

## Switching voice

```bash
# Warmer male voice — recommended if Daniel sounds too newsreader-y
VOICE="Oliver (Premium)" bash docs/voiceover/say_voiceover.sh

# Crisp female voice
VOICE="Kate" bash docs/voiceover/say_voiceover.sh

# Warmer female voice
VOICE="Serena (Premium)" bash docs/voiceover/say_voiceover.sh

# Slower delivery (default is 155 wpm; try 140-150 if it feels rushed)
RATE=145 bash docs/voiceover/say_voiceover.sh
```

If you don't have the Premium voices installed:

> System Settings → Accessibility → Spoken Content → System Voice →
> Manage Voices → tick **Oliver (Premium)** / **Serena (Premium)** / **Kate (Premium)**

They take a few minutes to download but the quality jump is significant —
the Premium voices are neural, much closer to a real BBC documentary cadence.

## After synthesis

1. Open your video editor (Final Cut Pro / iMovie / DaVinci Resolve / kdenlive).
2. Import the screencast you recorded against `docs/DEMO_STORYBOARD_v2.md`.
3. Drop each `out/sceneN.mp3` onto the audio track aligned with its scene.
4. If you want burned-in captions, import the scene `.txt` files as SRT
   (most editors accept plain-text via "Auto-generate captions" or manual
   timing).
5. Export 1080p H.264, AAC 128 kbps. Target ≤ 200 MB for Devpost upload.

## Why no music?

Quiet-luxury aesthetic. The terminal output + the narration carry the
weight. Background music on a 3-minute technical demo distracts from the
"this is rigorous engineering" signal that earns judge attention.

## Iteration loop

If a scene reads awkwardly when paired with the screencast:

1. Edit the corresponding `scenes/sceneN.txt`
2. Re-run `bash docs/voiceover/say_voiceover.sh` — it overwrites only the
   files that changed (no, it actually regenerates all of them; about 8
   seconds total)
3. Reimport that one MP3 into your editor

This keeps the source of truth (the text) version-controlled while the
audio rendering is reproducible.
